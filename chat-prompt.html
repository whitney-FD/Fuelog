Rules:
- Always wrap in an array [ ... ], even if sharing one recipe.
- Use the ingredients array — FuelLog auto-computes recipe totals from it and auto-saves each ingredient to my library for reuse.
- Per-ingredient cal / prot / carbs / fat / fiber / vita / iron / sugar / sodium are TOTALS for the qty × unit listed, not per unit.
  Example: 2 lb chicken at 480 kcal/lb → "qty": 2, "unit": "lb", "cal": 960.
- Allowed units: each, oz, lb, g, cup, tbsp, tsp, ml, serving.
- Units of measure: cal in kcal; prot / carbs / fat / fiber / sugar in grams; vita as % daily value; iron and sodium in mg.
- Use USDA FoodData Central values. Leave optional micronutrients (vita, iron, sugar, sodium) out of an ingredient rather than guessing.
- course must be one of: breakfast, main, side, snack, dessert, none. Anything else defaults to main.
- cats is an array of lowercase, hyphenated tags (e.g. "meal-prep", "one-pot"). Invent new ones freely.
- Do not include recipe-level macro totals — they're derived from the ingredient list.
- Do not include per-serving amounts — FuelLog divides totals by servings automatically.
- If the recipe has no web source, set "url": "".

Quick reference

  • Units: each, oz, lb, g, cup, tbsp, tsp, ml, serving
  • Courses: breakfast, main, side, snack, dessert, none
  • Ingredient macros are TOTALS for the qty — FuelLog converts to per-unit internally.
  • Wrap in [ ] even for one recipe.
